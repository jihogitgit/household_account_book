# 가계부 리서치 스킬

## 목표

최신 가계부 앱 트렌드와 UX 패턴을 조사하여 Streamlit 구현에 바로 적용 가능한 근거 기반 인사이트를 생성한다.

## 리서치 범위

### 한국 앱 (필수)
- **뱅크샐러드**: 자동 분류 정확도, 소비 패턴 분석 UI
- **토스**: 소비 리포트 화면, 월별 요약 방식
- **카카오뱅크**: 지출 분류 UX

### 글로벌 앱 (비교 참고)
- **YNAB**: 예산 배분 방식, 목표 설정 UI
- **Copilot** (iOS): AI 분류, 대시보드 레이아웃
- **머니포워드** (일본): 가계부 자동화, 자산 통합 화면

## 조사 방법

1. WebSearch로 각 앱의 최신 리뷰·스크린샷·기능 업데이트 검색
2. 앱스토어 리뷰에서 사용자가 언급하는 불만·요청 사항 수집
3. 발견된 기능을 Streamlit 구현 가능 여부로 필터링
4. 한국 사용자 맥락(토스뱅크 거래내역 구조, 공과금/경조사/자동이체) 반영

## 출력 형식

`_workspace/research_gaegabu.md`에 저장:

```markdown
# 가계부 앱 리서치 리포트
## 1. 경쟁 앱 비교 (기능 표)
## 2. 사용자가 가장 중요하게 여기는 기능 TOP 10
## 3. 효과적인 대시보드 UX 패턴
## 4. Streamlit 구현 우선순위 추천 (MVP vs 고도화)
## 5. 한국 사용자 특화 요소
## 6. 기술 제약 (Streamlit 버전, 컴포넌트 한계)
```

## 품질 기준

- 출처 다양화: 앱스토어 리뷰 + 기술 블로그 + 커뮤니티(디시인사이드, 클리앙, Reddit)
- 사용자 목소리 우선: 개발자 블로그보다 실제 리뷰 우선
- Streamlit 제약 현실화: 구현 불가 기능은 "Streamlit 외 도구 필요"로 명시
- 리포트 완성 후 planner에게 SendMessage로 완료 알림
